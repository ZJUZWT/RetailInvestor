# 散户大冒险 - 游戏设计文档

> 一个炒股主题的Roguelike网页游戏

## 1. 概述

### 1.1 核心概念

玩家扮演一名散户，通过每日交易一只股票来实现一个无厘头的人生目标（如"攒够50万开奶茶店"）。游戏以天为回合制，融合A股交易规则、随机事件系统和卡牌收集机制。

### 1.2 核心玩法定位

- **单股博弈模式**：玩家只操作一只股票，通过仓位管理来博弈
- **混合型Roguelike**：事件驱动（随机事件影响股价）+ 卡牌构筑（收集被动能力卡）
- **风格分级**：半写实（常见）→ 近似抽象/无厘头（稀有）→ 完全魔幻（传说），越离谱越稀有

### 1.3 胜负条件

- **胜利**：可用资产（现金 + 持仓市值）达到目标金额
- **失败**：可用资产不足以支付当天生活费（破产）
- **初始设定**：起始资金 10万，目标金额随机（50万-500万），每日生活费从 200元/天起

## 2. 回合结构

每一天（一回合）分为6个阶段：

### 2.1 晨报阶段（自动）

- 显示昨日收盘情况（涨跌幅、盈亏）
- 隔夜新闻/事件推送
- 可能触发事件链的后续发展
- 玩家被动接收信息，无需操作

### 2.2 上午交易（9:30-11:30）

- 查看K线图 + 今日开盘价
- 第一次交易决策：买入 / 卖出 / 观望
- 可调整买卖数量（仓位管理）
- 遵守A股规则：T+1（当日买入次日才能卖出）、涨跌停±10%

### 2.3 午间休息

- 获取信息环节（刷手机、看新闻、听小道消息）
- 固定消耗1点体力
- 信息可能是真实的也可能是误导性的
- 影响下午交易的判断依据

### 2.4 下午交易（13:00-15:00）

- 股价在上午基础上继续波动
- 第二次交易决策
- 尾盘效应：接近收盘时波动可能加剧
- 同样遵守A股规则

### 2.5 盘后活动（消耗体力）

玩家分配剩余体力到各项活动：

| 活动 | 体力消耗 | 效果 |
|------|---------|------|
| 📱 刷社交媒体 | 1 | 获取线索（可能是假的），影响情绪值 |
| 📊 复盘研究 | 1 | 获取较可靠的明日信息加成 |
| 🤝 社交应酬 | 2 | 可能获得内幕消息/卡牌，但要花钱 |
| 🎰 搞副业 | 1 | 赚/亏小钱（最后的翻盘机会或致命一击） |
| 😴 早点休息 | 0 | 明日体力+1 |

- 默认每日体力：3点（扣除午间1点后剩余2点）
- 某些活动可能带来额外开销（社交应酬花钱、副业亏损）
- 这是破产前的"最后机会"环节

### 2.6 今日结算（自动）

- 当日盈亏汇总
- 扣除生活费
- 资产检查：破产？达成目标？继续？
- 进入下一天

## 3. 事件系统

### 3.1 稀有度分级

| 稀有度 | 风格 | 概率 | 颜色标识 |
|--------|------|------|---------|
| 常见 | 半写实 | 60% | 白色 |
| 稀有 | 近似抽象/无厘头 | 30% | 蓝色 |
| 传说 | 完全魔幻 | 10% | 金色 |

### 3.2 事件效果类型

- **📈 利好**：增加股价上涨概率
- **📉 利空**：增加股价下跌概率
- **💰 财务影响**：直接影响现金或生活费
- **🃏 卡牌**：获得或失去能力卡
- **⚡ 体力影响**：增减体力值
- **🔗 事件链**：触发后续回合的连锁事件

### 3.3 事件示例

**常见（白色）- 半写实：**
- "央行宣布降准0.25%" → 小幅利好
- "公司发布季度财报，利润下滑" → 利空
- "板块资金流出，北向资金撤离" → 利空
- "国务院出台新政策支持该行业" → 利好

**稀有（蓝色）- 近似抽象：**
- "隔壁王大妈把养老钱全投进去了" → 散户情绪过热信号
- "CEO深夜发微博：明天见" → 不确定性增加
- "某大V在直播间喊单，弹幕刷满了" → 短期利好但可能是陷阱
- "你在电梯里听到两个西装男在聊这只股" → 获得线索

**传说（金色）- 完全魔幻：**
- "股神托梦给你：明天涨停" → 大概率利好（但也可能是反向指标）
- "一只金丝猴被任命为公司新任董事长" → 巨大不确定性
- "交易所服务器被外星人黑了，停牌一天" → 停牌
- "你在路边捡到一本《炒股秘籍》" → 获得传说卡牌

### 3.4 事件链机制

事件可以跨越多个回合发展：

```
Day N:   "公司CEO深夜发了一条意味深长的微博..."
Day N+1: "微博被删了，但截图已经传遍全网"
Day N+2: 分支 →
         A: "证监会宣布介入调查" → 重大利空
         B: "原来是CEO新书发布预告" → 小幅利好
         C: "CEO道歉：昨晚喝多了" → 微幅波动
```

事件链的结局可能受玩家行为影响（如是否在盘后活动中选择了"研究"）。

## 4. 卡牌系统

### 4.1 基础规则

- 卡牌 = 持久性被动能力（少数为主动，可手动触发）
- 通过事件和盘后活动获取
- 初始卡牌槽位：3个，最多可扩展到5个
- 卡牌可被事件夺走

### 4.2 卡牌稀有度

与事件系统对应：白色（常见）、蓝色（稀有）、金色（传说）

### 4.3 卡牌示例

| 名称 | 稀有度 | 类型 | 效果 | 获取途径 |
|------|--------|------|------|---------|
| 🔍 技术分析入门 | 白 | 被动 | 午间信息准确率+10% | 复盘研究 |
| 📺 抖音股神关注 | 白 | 被动 | 每天获得一条"大师分析"（50%是反向指标） | 刷社交媒体 |
| 🐀 老鼠仓门路 | 蓝 | 被动 | 每周一次提前知道明日涨跌方向 | 社交应酬 |
| 🧘 佛系持股心法 | 蓝 | 被动 | 涨停时不会触发"冲动追高"事件，跌停时不触发"恐慌抛售"事件 | 特殊事件 |
| 💎 钻石手 | 金 | 被动 | 连续持仓不卖的每一天，下次卖出收益+2%（累计） | 传说事件 |
| 🎯 涨停敢死队 | 金 | 主动 | 每局可使用一次：无视涨停限制买入/卖出 | 传说事件 |

## 5. 数值系统

### 5.1 玩家属性

| 属性 | 初始值 | 说明 |
|------|--------|------|
| 现金 | ¥100,000 | 可用于交易和生活 |
| 持仓 | 0股 | 当前持有的股票数量 |
| 体力 | 3/天 | 每日可分配到活动的点数 |
| 生活费 | ¥200/天 | 每日固定支出，可被事件改变 |
| 目标金额 | 随机 | ¥500,000 - ¥5,000,000 |

### 5.2 股价模拟

股价不是纯随机，有可读的规律：

1. **基础趋势**：每局随机生成隐藏的牛熊周期
   - 如：前30天慢牛(+0.5%/天基础) → 中间20天震荡(±0.2%) → 后面暴跌(-1%/天基础)
2. **事件修正**：利好/利空事件在基础趋势上叠加
3. **市场情绪**：根据连续涨跌天数动态调整波动率
   - 连涨5天 → 波动率增大（物极必反）
   - 连跌3天 → 可能触发"恐慌性抛售"或"抄底资金进场"
4. **A股规则**：
   - 涨停：当日涨幅不超过+10%
   - 跌停：当日跌幅不超过-10%
   - T+1：当日买入的股票次日才能卖出

### 5.3 难度曲线

- 生活费随天数缓慢增加（通货膨胀/生活品质提升）
- 事件烈度随天数增加（后期出现更多极端事件）
- 牛熊周期在后期更加剧烈

## 6. 技术架构

### 6.1 技术选型

| 层面 | 选择 | 理由 |
|------|------|------|
| 框架 | React 18 | 组件化开发，生态成熟 |
| 语言 | TypeScript | 类型安全，适合游戏逻辑 |
| 构建 | Vite | 快速开发体验 |
| 状态管理 | Zustand | 轻量，适合游戏状态 |
| K线图 | lightweight-charts | TradingView出品，轻量专业 |
| 样式 | Tailwind CSS | 快速UI开发 |
| 存档 | localStorage | 第一版无后端 |
| 部署 | Vercel / GitHub Pages | 免费静态托管 |

### 6.2 项目结构

```
src/
├── components/          # UI组件
│   ├── KLineChart/      # K线图组件
│   ├── TradingPanel/    # 交易面板
│   ├── EventDisplay/    # 事件展示（支持图片）
│   ├── CardCollection/  # 卡牌展示
│   ├── ActivityPanel/   # 盘后活动面板
│   └── StatusBar/       # 状态栏（资金/体力/天数）
├── engine/              # 游戏引擎
│   ├── GameEngine.ts    # 回合流程控制
│   ├── StockSimulator.ts# 股价模拟
│   ├── EventSystem.ts   # 事件系统
│   └── CardSystem.ts    # 卡牌系统
├── data/                # 游戏数据
│   ├── events/          # 事件配置JSON
│   ├── cards/           # 卡牌配置JSON
│   └── goals/           # 目标配置JSON
├── stores/              # Zustand状态

│   └── images/          # 事件配图等
├── utils/               # 工具函数
└── App.tsx              # 入口
```

### 6.3 核心数据模型

```typescript
// 游戏状态
interface GameState {
  day: number;
  phase: 'morning_news' | 'am_trading' | 'lunch_break' | 'pm_trading' | 'after_hours' | 'settlement';
  cash: number;
  shares: number;
  dailyExpense: number;
  stamina: number;
  maxStamina: number;
  goal: Goal;
  cards: Card[];
  maxCardSlots: number;
  activeEventChains: EventChain[];
  stockHistory: StockDataPoint[];
  gameStatus: 'playing' | 'won' | 'lost';
}

// 股票数据点
interface StockDataPoint {
  day: number;
  open: number;
  close: number;
  high: number;
  low: number;
  volume: number;
}

// 事件
interface GameEvent {
  id: string;
  title: string;
  description: string;
  imageUrl?: string;
  rarity: 'common' | 'rare' | 'legendary';
  effects: EventEffect[];
  chainNext?: string[];  // 后续事件ID
}

// 卡牌
interface Card {
  id: string;
  name: string;
  description: string;
  rarity: 'common' | 'rare' | 'legendary';
  type: 'passive' | 'active';
  effects: CardEffect[];
  usesRemaining?: number;  // 主动卡使用次数
}

// 目标
interface Goal {
  title: string;         // "攒够50万开奶茶店"
  targetAmount: number;
  description: string;
}
```

## 8. 第一版范围（MVP）

### 包含

- 完整回合流程（6个阶段）
- 基础股价模拟（趋势 + 随机 + 事件修正）
- K线图渲染
- 15-20个常见事件 + 5-8个稀有事件 + 2-3个传说事件
- 1-2条事件链
- 6-8张卡牌
- 3个无厘头目标模板
- 基础盘后活动（5种）
- 本地存档
- 桌面端 + 移动端基础适配

### 不包含（后续迭代）

- 后端服务/云存档
- 排行榜/社交分享
- 更多事件/卡牌/目标
- 音效/BGM
- 多股票/黄金坑位
- 对战模式
