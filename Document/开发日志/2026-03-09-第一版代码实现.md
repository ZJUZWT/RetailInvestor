## AI的实现思考

### 架构决策

选择了**"引擎-状态-视图"三层分离**的架构：

- `engine/` 层是纯函数，不依赖React，方便测试和复用
- `stores/` 层通过Zustand将引擎逻辑包装为响应式状态
- `components/` 层纯粹消费状态和调用actions

这种分离的好处是：如果将来要换UI框架（比如做移动端），引擎层完全不用动。

### 股价模拟算法

最终采用的方案是**趋势段 + 随机波动 + 事件修正**：

1. 每局预先生成3-5个趋势段（牛/熊/震荡），有倾向交替的倾向
2. 每天的价格变化 = 趋势偏差 + 随机波动 + 事件效果
3. 涨跌停硬限制在±10%

这个方案的好处是：玩家如果认真"看盘"可以感知到趋势，但不会完全可预测。事件系统提供了额外的变数，让每次体验都不同。

### 关于上下午双盘的实现

一开始考虑用两个完全独立的K线点来表示上下午。但这会让K线图数据量翻倍且不符合真实K线惯例。最终方案是：

- 上午生成 `open → amClose`
- 下午在amClose基础上继续波动生成最终 `close`
- K线图只显示每天一根完整的日K线

交易面板在两个时段都可操作，但价格在上午用的是开盘价附近，下午用的是接近收盘价。这样既有两次决策机会，又不会让K线图过于复杂。

### 事件文案创作

这次一口气写了32个事件，覆盖三个风格等级。一些设计心得：

- **常见事件**偏真实，用了很多A股股民熟悉的场景（降准、北向资金、大股东减持）
- **稀有事件**引入社会化元素（王大妈、出租车司机、同学聚会）
- **传说事件**走魔幻路线，有一定幽默效果

事件链只做了一条（CEO微博系列），但机制已经搭好，后续添加很容易。

### 盘后活动的随机性设计

每个活动都有随机结果，关键设计点：

- **社交媒体**：35%概率给出正确方向，65%是噪音。这模拟了社交媒体信息质量差的现实。
- **研究分析**：65%概率给出正确方向。比社交媒体靠谱，但不是100%。
- **社交应酬**：花500-2000元，20%概率获得卡牌。高风险高回报。
- **搞副业**：随机正负收入，是最有戏剧性的活动（可能救命也可能致命）。
- **休息**：稳定收益，明天多1点体力。风险厌恶者的选择。

### 遇到的技术问题

1. **lightweight-charts API变化** - v5版本的API和v4不同，`addSeries(CandlestickSeries, ...)` 替代了旧的 `addCandlestickSeries()`
2. **Tailwind CSS 4** - 使用了新的 Vite 插件方式，不再需要 PostCSS 配置
3. **Zustand `set` 的partial更新** - 需要注意set只做浅合并
