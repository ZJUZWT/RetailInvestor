## 1. 分时图交易点位标记 (B / S / T)

在分时图上标注玩家的交易操作位置：

- **B (Buy)** - 买入点，红色标记
- **S (Sell)** - 卖出点，绿色标记
- **T (T+1)** - 今日买入、次日卖出的标记（区分隔夜持仓卖出 vs 当日T+1卖出）

### 实现思路

1. **Store 层**: buy/sell action 中记录交易发生时的 `currentTick`（分钟序号），存入一个 `tradeMarkers` 数组：
   ```ts
   type TradeMarker = { tick: number; price: number; type: 'B' | 'S' | 'T' };
   ```
   - 每天开盘时清空（或保留最近几天用于5日分时图）
   - T 标记：当 boughtToday=true 且次日卖出时标记为 T

2. **KLineChart 组件**: 使用 lightweight-charts 的 `markers` API 在分时线上绘制标记点
   ```ts
   series.setMarkers([
     { time: tick as UTCTimestamp, position: 'belowBar', color: '#ef4444', shape: 'arrowUp', text: 'B' },
     { time: tick as UTCTimestamp, position: 'aboveBar', color: '#22c55e', shape: 'arrowDown', text: 'S' },
   ]);
   ```

## 2. 分时图右侧价格轴显示涨跌百分比及颜色

当前分时图右侧的价格刻度是白色的绝对价格，需要改为同时显示对应的涨跌幅百分比，并用红涨绿跌的颜色。

### 实现思路

1. 使用 lightweight-charts 的 `priceFormat` 或自定义 `priceFormatter` 回调：
   ```ts
   rightPriceScale: {
     borderColor: '#1a1a2e',
   }
   // 在 series 上设置:
   priceFormat: {
     type: 'custom',
     formatter: (price: number) => {
       const pct = ((price - todayOpen) / todayOpen * 100);
       const sign = pct >= 0 ? '+' : '';
       return `${price.toFixed(2)}  ${sign}${pct.toFixed(2)}%`;
     },
   }
   ```

2. 颜色方面：lightweight-charts 的 rightPriceScale 不直接支持按值着色，可能需要：
   - 方案A: 用 `priceFormatter` 只显示格式化文本，颜色靠默认的 series 颜色（白色线 = 白色轴标签）
   - 方案B: 创建一个不可见的辅助 series 来控制最后价格标签的颜色
   - 方案C: 用 CSS overlay 覆盖在价格轴区域上方，手动渲染带颜色的百分比

3. 调研 lightweight-charts v5 是否有 `priceScaleTextColorProvider` 或类似 API，如没有则用方案B或C。
