# 散户大冒险 - 项目记忆

> 最后更新: 2026-03-10

## 项目概述

炒股主题的Roguelike网页游戏。玩家扮演散户，每日交易一只A股股票，在随机事件中实现无厘头人生目标（如开奶茶店、给丈母娘买房）。

## 核心设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 核心模式 | 单股博弈 | 简化决策空间，聚焦仓位管理 |
| Roguelike机制 | 混合型（事件+卡牌） | 事件驱动为主，卡牌构筑为辅 |
| 死亡机制 | 多种死法：破产/猝死/饿死/精神崩溃 | 精力系统增加生存压力 |
| 风格调性 | 三级混合（写实→无厘头→魔幻） | 越离谱概率越低 |
| 回合结构 | 晨报→上午盘→午休→下午盘→盘后→结算 | 贴合A股双盘 |
| 生命系统 | 精力+饥饿+SAN值三属性 | 时间连续流动，角色管理 |
| 技术栈 | React 18 + TS + Vite + Zustand + lightweight-charts + Tailwind | K线生态好 |

## 回合流程

1. **晨报** - 显示隔夜新闻/事件
2. **上午交易(9:30-11:30)** - **实时分时图**动态流动，玩家随时可买卖
3. **午间休息** - 分时图凝固，可切换日K/周K/月K分析（不消耗体力）
4. **下午交易(13:00-15:00)** - 继续实时分时图，随时交易
5. **盘后活动** - 消耗体力做活动（刷社交/研究/应酬/副业/休息）
6. **今日结算** - 扣生活费，检查胜负

### 生命体征系统（v3新增）

三大属性管理角色生存：
- **⚡精力(0-100)**: 清醒时-4/h，睡觉恢复（非线性：越多越好），归零=猝死
- **🍚饱腹(0-100)**: 每小时-4.5，吃东西恢复(4种食物花钱买)，归零=饿死
- **🧠SAN值(0-100)**: 亏损/看跌盘降SAN(看盘时2.5x)，睡觉+赚钱恢复，<20失控(30%阻止交易)，=0精神崩溃

食物系统(v1简单版): 方便面(¥8/+25)、零食(¥15/+15)、外卖(¥30/+50)、下馆子(¥80/+75)

睡觉机制：选择时长→暂停玩家操作→时间加速→精力非线性恢复

### 日历系统（v3新增）

- 游戏从2026-01-05(周一)09:00开始
- 周历组件显示交易日/休市日
- totalGameMinutes驱动日历计算

### 分时图增强（v3新增）

- 交易点位标记(B/S)：使用createSeriesMarkers API
- 价格轴涨跌百分比显示：自定义priceFormatter

## 技术架构

```
src/
├── types/         # 核心类型（GameState, GameEvent, Card等）
├── data/          # 游戏数据（events.ts, cards.ts, goals.ts, activities.ts）
├── engine/        # 纯逻辑引擎（StockSimulator, IntradaySimulator, EventSystem, CardSystem）
├── stores/        # Zustand状态管理（gameStore.ts - 含实时tick播放）
├── components/    # React UI组件（12个）
└── App.tsx        # 主入口
```

引擎层是纯函数，不依赖React。Store层包装引擎为响应式状态+定时器播放。UI层消费状态。

## 股价模拟

- 每局预生成3-5个趋势段（牛/熊/震荡），倾向交替
- **日内分时**: 预生成241tick/天, gravity+noise+trendPush+momentum算法
- **8时段特征**: 不同波动率/趋势跟随/动量衰减
- A股规则: 涨跌停±10%, T+1

## 内容量

- 事件: 18常见 + 10稀有 + 4传说 = 32个
- 卡牌: 4白 + 3蓝 + 1金 = 8张
- 目标: 5个
- 活动: 5种
- 事件链: 1条（CEO微博系列）

## 用户偏好

- 用户不熟悉前端技术，技术决策由AI做
- 用户重视图片对无厘头效果的增强
- 用户提出事件有连续性（事件链）
- 用户希望游戏有趣且易于分享
- 用户改进了回合结构（添加上下午双盘、盘后活动在结算前）
- 用户希望盘后活动既有休闲也有影响后续的策略性活动
- 用户提出了体力系统的概念
- 用户要求盘中实时分时图体验（v2核心需求）
- 用户要求盘中速度可调节
- 用户要求休市后可切换日K/周K/月K/5日线查看
